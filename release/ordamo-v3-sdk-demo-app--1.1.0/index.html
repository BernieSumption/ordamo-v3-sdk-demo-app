Ordamo SDK Demo App

Welcome to the SDK demo app. This page demonstrates the major features of the SDK.

Introduction

The Ordamo V3 system is a ceiling-mounted device that projects an interactive surface onto a table. This is used simultaneously by several people, for example at the "inamo" series of interactive restaurants.

The frontend software running on the V3 units is a modular HTML5 application. A wrapper program called the apphost loads and displays apps. Each app is simply an HTML page.

The Ordamo V3 SDK is a package designed to ease the process of developing apps. In particular, it allows apps to:

  • declare what kind of content the app requires - this is used to create a CMS interface for the app
  • load content from the CMS server at runtime
  • find out where plates are positioned on the table in order to avoid drawing content in places where it will be covered by a plate
  • save state, so that if a user leaves the app they can resume where they left off on returning
  • during development, mock out the above features so that developers can test an app without access to a V3 unit or CMS server

Getting started

This file you're looking at right now is a V3 app. It's built using Brunch because Brunch is a great built tool for small apps. The thing that makes it a V3 app is its use of the ordamo-v3-sdk NPM package, defined as a dependency in package.json. You don't have to use Brunch, or even node.js to make a V3 app - but we recommend that apps are created by copying this demo app template. It's a pattern that we've found to work well.

The best way to get started is to read through the source code of this app. It's well commented.

You may find these scripts useful, run them from a command window in the same folder as the app's package.json:

  • npm start - launches a development server at http://localhost:3333/ that serves this documentation page. It watches your source folder and whenever you change anything it recompiles the code and refreshes the browser window
  • npm run release - make a release build. The newly created build file is copied to release/app-name--version.js
